记忆系统架构设计:
  记忆系统分为三层。
  会话记忆:
    范围:当前对话
    持久性:仅存在于当前对话
    存储:浏览器indexdb → Redis →(会话结束) postgresql 
    具体实现:
      - 最低触发阈值: 10k token，当前对话上下文窗口达到10k token才激活。
      - 更新阈值: 第一次更新之后，每增长 5k tokens 更新一次。
    会话记忆摘要文档模板:`session_memory_template.md`
    大小限制: 每个章节最大 2k tokens,整个文件最大12k tokens
    自定义:用户可以更改模板和提示词。
    压缩机制: 
    - 会话摘要式压缩: 超过三次5k token对摘要进行更新失败后改为增长10k token对摘要更新一次。只要10k token的那次更新成功，就回到5k token对摘要更新。
    - 传统全量压缩: 计算token，如果当前上下文窗口的token达到当前模型最大上下文窗口的80%，则进行传统全量压缩。全量压缩提示词模板`full_compact_template.md`
    
  持久记忆:
  范围:跨对话
  持久性:永久，除非手动删除
  存储:postgresql、redis
  记忆类型分为三种[user,feedback,project]:
    - user: 用户角色，目标、知识背景等用户的身份信息，当了解到用户的身份信息时进行保存。
    - feedback: 用户纠正或确认做法时保存，用户对工作方式的指导。
    - project: 用户当前追踪的主题与事件进展
  建立一个MEMORY文档专门用于进行记忆索引。
    格式说明: 每一行应该是一条持久记忆的说明，作为索引，如果涉及到，AI可以调用工具从数据库读取具体的记忆。
    具体格式:
      - [记忆类型-记忆标题] - 记忆简短总结(50token以内)
    示例:
      - [feedback-concise] - 不在回复末尾加回顾性总结 

  核心记忆:
    范围:跨对话
    持久性:永久，并且无法删除，只能更改
    存储:postgresql
    内容: agent工作的核心提示词，必须常驻长期向AI发送，还有可调用工具列表和说明。
    摘要文档和进行摘要的提示词，工具的详细使用说明也在此范畴，但是它们是因为某种条件触发的。
